# Slider Component

Рефакторенный компонент слайдера с улучшенной архитектурой и разделением ответственности.

## Структура

```
components/Slider/
├── Slider.tsx                 # Главный компонент
├── Slider.module.scss         # Основные стили контейнера
├── components/                # Подкомпоненты
│   ├── SlideItem/            # Отдельный слайд
│   │   ├── SlideItem.tsx
│   │   └── SlideItem.module.scss
│   ├── NavigationArrows/     # Стрелки навигации
│   │   ├── NavigationArrows.tsx
│   │   └── NavigationArrows.module.scss
│   ├── Pagination/           # Точки пагинации
│   │   ├── Pagination.tsx
│   │   └── Pagination.module.scss
│   └── index.ts              # Экспорт подкомпонентов
├── hooks/                     # Кастомные хуки
│   ├── useAutoScroll.ts      # Логика автопрокрутки
│   ├── useSliderScroll.ts    # Логика управления скроллом
│   ├── useMediaQuery.ts      # Определение размера экрана
│   └── index.ts              # Экспорт хуков
└── config/                    # Конфигурация
    └── slides.tsx             # Данные слайдов
```

## Ключевые улучшения

### 1. **Разделение на подкомпоненты**
- `SlideItem` - отвечает за отрисовку одного слайда
- `NavigationArrows` - управление стрелками навигации
- `Pagination` - отображение точек пагинации

### 2. **Кастомные хуки**
- `useAutoScroll` - инкапсулирует логику автоматической прокрутки
- `useSliderScroll` - управляет состоянием прокрутки и навигацией
- `useMediaQuery` - определяет размер экрана для адаптивности

### 3. **Новые возможности**

#### Оверлей с затемнением
Добавлен полупрозрачный оверлей поверх изображений для улучшения читаемости текста:
```scss
.overlay {
  background-color: rgba(0, 0, 0, 0.25);
}
```

#### Адаптивные изображения
Для экранов меньше 768px можно задать отдельные изображения:
```tsx
{
  id: "first-visit",
  desktopImage: "img/slide-1.webp",
  mobileImage: "img/slide-1-sm.webp",  // Отдельная картинка для мобильных
  content: <...>
}
```

### 4. **Упрощенные стили**
- Стили разделены по компонентам
- Убраны дублирующиеся правила
- Улучшена читаемость и поддерживаемость

## Использование

Главный компонент остается простым и декларативным:

```tsx
import Slider from "@/components/Slider/Slider";

<Slider />
```

## Добавление новых слайдов

Редактируйте файл `config/slides.tsx`:

```tsx
export const slides: SlideConfig[] = [
  {
    id: "unique-id",
    desktopImage: "path/to/desktop-image.webp",
    mobileImage: "path/to/mobile-image.webp",
    content: (
      <h1>Ваш контент</h1>
    ),
  },
  // ...другие слайды
];
```

## Настройка автопрокрутки

В файле `Slider.tsx` можно настроить параметры:

```tsx
const { resetAutoScrollTimer, isAutoScrollingRef } = useAutoScroll({
  sliderRef,
  intervalMs: 1000,  // Частота проверки (мс)
  delayMs: 5000,     // Задержка перед автопрокруткой (мс)
});
```

## Преимущества рефакторинга

✅ **Модульность** - каждый компонент отвечает за одну функцию  
✅ **Переиспользуемость** - хуки можно использовать в других компонентах  
✅ **Тестируемость** - изолированные компоненты легче тестировать  
✅ **Читаемость** - код разбит на логические части  
✅ **Масштабируемость** - легко добавлять новые функции  
✅ **Адаптивность** - поддержка разных изображений для мобильных и десктопов  
✅ **Улучшенный UX** - оверлей повышает читаемость текста
